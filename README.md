# 中文学术论文文献综述助手

面向中文论文写作的 Codex Skill。你可以先导入已有文献；如果手里还没有文献，它也可以根据论文主题查找公开来源中的候选文献，生成中文数据库检索策略、候选文献矩阵、初步文献综述、研究脉络、研究空白和创新点建议。

> 不编造参考文献，不绕过数据库权限，不直接替代学术判断。这个项目的目标是让中文论文写作先回到“真实文献”上。

一句话理解：**没有文献也能先开始，有文献就优先导入；所有公开来源候选文献先标为“待核验”，核验后再进入正式论文内容。**

## 它解决什么问题

很多人在用 AI 写文献综述时会遇到三个问题：

- 参考文献容易被编造，作者、期刊、年份对不上。
- 综述段落像模板，只有“研究具有重要意义”，缺少具体文献脉络。
- 创新点很空泛，不知道自己到底还能从哪个角度写。

这个 skill 的工作方式相反：优先导入你已有的 CNKI、万方、维普、学校图书馆、期刊官网、Zotero、BibTeX、RIS 或 PDF 文献资料；如果你还没有文献，它会先帮你拆关键词、生成数据库检索式、推荐查找路径，并从 OpenAlex、Crossref 等公开来源补充候选文献。在候选文献元数据足够时，它可以先生成“待核验版”的文献矩阵和综述段落，等你核验后再升级为正式论文内容。

## 适合谁

- 正在写本科毕业论文、课程论文、开题报告的学生
- 正在写硕士论文、博士论文、期刊论文的研究者
- 需要整理中文文献的科研助理、课题组成员
- 想把 Zotero、CNKI、万方、维普题录转成文献矩阵的人
- 需要分析研究空白、创新点和投稿方向的人

## 不适合什么场景

- 代写整篇论文或替代作者完成学术判断
- 伪造参考文献、作者、期刊、年份、DOI 或研究结论
- 绕过 CNKI、万方、维普、学校图书馆的登录、付费、验证码或机构权限
- 承诺查重、AI 检测、期刊录用或导师认可
- 把公开来源候选文献直接当作已经核验的正式引用

## 你可以用它做什么

| 功能 | 输出 |
| --- | --- |
| 从零查找文献 | 在用户没有文献时，根据主题生成检索词，并从公开来源补充候选文献 |
| 初步文献综述 | 基于公开来源候选文献生成待核验版综述段落、研究脉络和研究空白，不能直接当正式引用 |
| 补充候选文献 | 当已有文献不足时，使用 OpenAlex、Crossref 等公开来源补充待核验候选文献 |
| 文献导入 | 识别 CNKI、万方、维普、Zotero、BibTeX、RIS、PDF 摘要和复制题录 |
| 首批文献主张图 | 用户首次上传文献或从公开来源找到候选文献后，列出作者+年份+一句话核心主张，按共同假设分组，标记矛盾论文，并绘制宏观背景图 |
| 文献矩阵 | 作者、年份、题名、期刊、摘要、关键词、方法、结论、局限 |
| 研究脉络 | 谁较早提出、谁可能延伸谁、哪些研究存在对比或补充关系 |
| 研究领域知识地图 | 输出核心主张、3-5 个支持柱、2-3 个争议区、1-2 个前沿问题和初学者必读论文 |
| 研究空白 | 当前文献中尚未充分解决的问题 |
| 创新点建议 | 3-5 个可写方向，并标记最接近论文、相近之处、可推进空间、推荐方法论和核验风险 |
| 三种输出模式 | 按本科论文、硕士论文、期刊论文调整文献深度、综述结构和创新点标准 |
| 综述写作 | 生成可放入引言、文献综述或开题报告的中文段落 |
| 去 AI 化表达 | 减少模板句，保留事实、引用和学术语气 |
| 期刊推荐 | 给出中文期刊匹配思路和人工核验提醒 |
| 完整中文工作流演示 | 展示从“没有文献”到“检索策略、候选文献、矩阵、知识地图、创新点、方法论、去 AI 化综述”的端到端效果 |

如果你只想先跑通一次，可以直接复制更短版本：

```text
使用 chinese-academic-paper-assistant。我的论文主题是【填写研究方向】。我目前没有整理好的文献，请先帮我生成 CNKI、万方、维普、学校图书馆和期刊官网的检索策略，再基于可公开核验的候选文献输出待核验版文献矩阵、首批文献主张表、研究脉络、知识地图、研究空白、创新点、最接近论文和方法论建议。所有未核验文献请标记为“待核验”，不要编造参考文献，不要把待核验文献写成正式引用。
```

## 安装方式

把 skill 文件夹复制到你的 Codex skills 目录：

```powershell
mkdir $env:USERPROFILE\.codex\skills -Force
Copy-Item -Recurse .\chinese-academic-paper-assistant $env:USERPROFILE\.codex\skills\
```

然后在 Codex 中使用：

```text
使用 chinese-academic-paper-assistant，帮我整理这些中文文献，生成文献矩阵、研究脉络、创新点和去 AI 化综述段落。
```

如果你的 Codex 支持从 GitHub 安装 skill，可以使用这个仓库路径：

```text
https://github.com/qinky1234-sys/chinese-academic-paper-skill/tree/main/chinese-academic-paper-assistant
```

## 推荐工作流

1. 如果你已经有文献，准备 8-15 条真实文献题录。
2. 如果你还没有文献，先让 skill 根据论文主题从公开来源查找候选文献，并生成 CNKI、万方、维普、学校图书馆、期刊官网的检索策略。
3. 优先导入 Zotero、BibTeX、RIS、CNKI、万方、维普、学校图书馆或期刊官网导出的资料。
4. 先生成文献矩阵，不急着写正文。
5. 检查作者、题名、期刊、年份、摘要、关键词、DOI/链接是否完整。
6. 文献不够时，再用公开来源或已配置接口补充候选文献；候选文献可以生成待核验版综述，但必须核验后再进入正式论文正文。
7. 在矩阵基础上分析共同点、研究脉络、研究空白和创新点；每个创新点都要附最接近论文和推荐方法论。
8. 最后生成去 AI 化中文综述段落。

## 三种输出模式

| 模式 | 适合场景 | 输出重点 |
| --- | --- | --- |
| 本科论文模式 | 本科毕业论文、课程论文、初次写综述 | 主题分类清楚、概念解释稳妥、创新点小而可做 |
| 硕士论文模式 | 硕士论文、开题报告、研究设计前期 | 研究脉络、理论/方法差异、研究空白、创新点可行性 |
| 期刊论文模式 | 投稿论文、论文压缩投稿、文章引言 | 问题意识、文献争议、贡献表达、期刊匹配提醒 |

使用时可以直接说：

```text
请按硕士论文模式输出：先生成文献矩阵，再分析研究脉络、研究空白和创新点，最后写一段去 AI 化中文综述。
```

## 支持的导入来源

即使你暂时没有文献，也可以先输入论文主题、研究方向、关键词、学科和论文类型，让 skill 帮你查找公开候选文献、生成数据库检索路径，并产出待核验版文献综述。

- CNKI、万方、维普导出的题录或引用信息
- 学校图书馆数据库导出的文献记录
- 期刊官网公开题录
- Zotero、EndNote、NoteExpress
- BibTeX、RIS、DOI 列表
- PDF 摘要、复制的参考文献、手工整理的文献表

导入字段越完整，输出越可靠。推荐至少包含：作者、题名、期刊或来源、发表年份、摘要、关键词、DOI/URL 或数据库来源。

## 学术边界

这个 skill 不会做这些事：

- 编造参考文献、作者、期刊名、发表时间或研究结论
- 绕过 CNKI、万方、维普、学校图书馆的登录、付费、验证码或权限限制
- 虚构核心、CSSCI、CSCD、影响因子、审稿周期、版面费或录用率
- 承诺通过查重、AI 检测或期刊录用
- 把字段不完整、来源不可追溯的文献直接写进正式综述

它可以帮助你整理、分析和改写，但最终引用核验、学术判断和论文责任仍然需要由作者完成。

## 防止编造文献的规则

- 用户导入的文献优先，公开来源只作为补充线索。
- 公开来源候选文献必须标注为“待核验”。
- 不确定作者、年份、期刊、来源、DOI 或 URL 时，不能写成正式引用。
- 没有引用证据时，不能断言“谁引用了谁”，只能写“可能存在承接关系，需进一步核验”。
- 创新点只能说“在当前文献范围内较少发现”，不能说“首次提出”“无人研究”“绝对创新”。
