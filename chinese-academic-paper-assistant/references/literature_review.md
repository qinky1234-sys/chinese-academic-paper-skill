# Literature Review

## One-Click Review Mode

Use this mode when the user wants 一键生成文献综述, 一键综述, 直接生成综述, 只要最终综述, or 不用其他功能.

Output only:

1. 完整版文字式文献综述
2. 文献覆盖简表
3. 必要核验提醒

Do not show the full matrix, knowledge map, innovation points, journal matching, or long process notes unless the user requests them. Recommend 8+ usable sources; if fewer are available, generate the draft but state that coverage is limited.

## Matrix Fields

Use these columns by default:

| 编号 | 作者 | 年份 | 题名 | 期刊/来源 | 发表时间 | 研究对象 | 研究方法 | 核心观点 | 局限性 | 与本文关系 | 来源状态 |
|---|---|---|---|---|---|---|---|---|---|---|---|

Do not place a source in formal review paragraphs unless 作者、题名、期刊/来源、发表时间、来源状态 are available.

## Grouping Methods

Choose one dominant organization and say why:

- By topic: useful for broad or interdisciplinary topics.
- By timeline: useful when the user asks for research development.
- By method: useful when methods differ strongly.
- By object/scenario: useful for policy, management, education, regional, or industry topics.
- By controversy: useful when findings disagree.

## Review Paragraph Requirements

Chinese review paragraphs should:

- In one-click mode, cover every uploaded or found source at least once.
- Use expressions like `XXX学者在《xxxxx》一文中提出/指出/认为……`.
- Begin from a concrete research problem rather than a generic era statement.
- Move from established work to unresolved problems.
- Cite each claim that depends on literature.
- Preserve uncertainty when evidence is thin.
- Avoid copying source sentences.
- Avoid inflated claims such as "填补国内外空白" unless the evidence is strong and verified.

## Citation Patterns

Use the user's requested style. If unspecified, use author-year Chinese academic style:

- `张三等（2023）认为...`
- `已有研究从治理结构、技术工具和主体协同三个角度展开讨论（李四，2022；王五，2024）。`

For numbered style, keep citation numbers stable after the user confirms the matrix.
