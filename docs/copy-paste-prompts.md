# 复制即用提示词模板

把下面模板中的【】内容替换成自己的主题、方向或文献资料即可使用。建议每次只做一个明确任务，先整理文献，再写综述。

## 1. 第一次使用：导入文献并生成矩阵

```text
使用 chinese-academic-paper-assistant。

我的论文主题是【填写论文主题】。
研究方向是【填写研究方向】。
论文类型是【本科论文/硕士论文/期刊论文】。

下面是我已经找到的文献资料，可能来自 CNKI、万方、维普、Zotero、BibTeX、RIS 或 PDF 摘要。
请你先识别资料格式，再整理成文献矩阵。

要求：
1. 检查作者、题名、期刊/来源、发表时间、摘要、关键词、DOI/链接是否完整。
2. 字段不完整的文献放入“待核验文献”，不要写进正式综述。
3. 不要编造文献、结论、期刊信息或发表时间。
4. 最后告诉我还需要补充哪些字段。

【在这里粘贴文献资料】
```

## 2. Zotero/BibTeX/RIS 导入

```text
使用 chinese-academic-paper-assistant。

下面是我从 Zotero 或数据库导出的 BibTeX/RIS 文献。
请先判断它是 BibTeX、RIS 还是混合格式，然后解析成文献矩阵。

请输出：
1. 已识别的导入格式
2. 成功解析的文献数量
3. 缺失字段清单
4. 可进入正式综述的文献
5. 需要人工核验的文献
6. 下一步是否需要补充检索

【在这里粘贴 BibTeX 或 RIS 内容】
```

## 3. 没有文献或文献不够：查找候选文献并生成初步综述

```text
使用 chinese-academic-paper-assistant。

我的论文主题是【填写主题】。
研究方向是【填写方向】。
我目前已有文献数量是【填写数量/0】篇。

请先根据主题从公开来源查找候选文献，同时生成可用于 CNKI、万方、维普和学校图书馆的检索策略。

要求：
1. 给出核心关键词、同义词和扩展词。
2. 给出 3-5 组中文高级检索式。
3. 说明每组检索式适合找哪类文献。
4. 告诉我导出时应该保留哪些字段。
5. 如果公开来源候选文献的元数据足够，请生成待核验版文献矩阵和初步综述。
6. 待核验文献不要当成正式可引用文献。
7. 不要绕过登录、付费、验证码或学校权限。
```

## 4. 从文献矩阵生成文献综述

```text
使用 chinese-academic-paper-assistant。

下面是我已经整理好的文献矩阵。
请基于这些文献生成一段可以放入论文【引言/文献综述/开题报告】中的中文综述。

要求：
1. 先概括文献共同点。
2. 再说明研究脉络：谁较早提出，谁可能延伸或回应了前人的研究。
3. 只有在引用数据明确时，才说“谁引用了谁”。
4. 没有证据时，用“从发表时间和研究内容看，可能存在承接关系，需进一步核验”。
5. 文字要自然，减少 AI 味，不要使用空泛套话。
6. 保留作者和年份引用标记。
7. 不要添加矩阵中没有的文献或结论。

【在这里粘贴文献矩阵】
```

## 5. 分析研究空白和创新点

```text
使用 chinese-academic-paper-assistant。

请根据下面的文献矩阵，分析这个研究方向中我可以做的创新点。

要求：
1. 先总结这些文献的共同研究对象、理论、方法和结论。
2. 再分析已有研究还没有充分解决的问题。
3. 用谨慎表达，不要直接说“没人做过”。
4. 给出 3-5 个可选创新点。
5. 每个创新点都要说明：已有基础、未解决问题、我可以怎么做、需要什么数据或材料、适合放在论文哪个章节。
6. 标出哪些创新点适合本科论文，哪些适合硕士论文或期刊论文。

【在这里粘贴文献矩阵】
```

## 6. 去 AI 化中文综述段落

```text
使用 chinese-academic-paper-assistant。

请把下面这段文献综述改得更像中文论文中自然写出的段落。

要求：
1. 保留事实、引用和逻辑顺序。
2. 不新增文献，不新增数据，不虚构结论。
3. 减少“随着时代的发展”“具有重要意义”“综上所述可以看出”等模板句。
4. 让句子长短更自然，段落衔接更顺。
5. 不承诺绕过 AI 检测，只做学术表达润色。

【在这里粘贴原文】
```

## 10. 完整工作流

1. 识别并整理我提供的文献资料。

边界：
不要编造参考文献。
不要把待核验文献写进正式综述。

## 使用建议

- 文献越真实、字段越完整，输出越可靠。
- 先让 skill 做矩阵，再让它写综述。
- 对“谁引用了谁”保持谨慎，除非你提供了明确引用数据。
- 对 CNKI、万方、维普、期刊官网、学校图书馆中的实时信息，最终仍要人工核验。
