# 常见问题 FAQ

这份 FAQ 面向第一次使用 `chinese-academic-paper-assistant` 的用户，重点解释文献来源、数据库边界、待核验文献、输出质量和 Pro 版方向。

## 1. 没有文献也能用吗

可以。

如果你现在还没有整理好的文献，可以先输入论文主题、研究方向、学科、论文类型和关键词。skill 会先做三件事：

- 拆解核心关键词、同义词和扩展词。
- 生成 CNKI、万方、维普、学校图书馆和期刊官网的检索策略。
- 从 OpenAlex、Crossref、期刊官网公开页面等公开来源补充候选文献。

如果候选文献的作者、题名、年份、来源、摘要或链接等元数据足够，skill 可以先生成“待核验版”文献矩阵和初步综述。正式写入论文前，仍然需要你回到数据库或期刊官网核验。

详细操作可以看：[没有文献时怎么用](./no-literature-workflow.md)。

## 2. 候选文献能直接写进论文吗

不建议直接写进正式论文。

候选文献适合用于前期选题、扩展关键词、判断研究方向和搭建初步文献框架。正式综述应只使用你确认过的文献，至少要核验：

- 作者是否正确
- 题名是否正确
- 期刊或来源是否正确
- 发表年份或发布时间是否正确
- DOI、URL 或数据库来源是否能追溯
- 摘要、研究方法和结论是否与原文一致

如果没有证据证明两篇文献之间存在引用关系，skill 只会用“可能存在承接关系，需进一步核验”这类表述，不会直接断言“谁引用了谁”。

## 3. CNKI、万方、维普可以自动查吗

开源版不会自动登录或抓取 CNKI、万方、维普，也不会绕过付费、验证码、学校权限或数据库限制。

推荐做法是：

1. 你在 CNKI、万方、维普或学校图书馆中合法检索。
2. 使用数据库自带的“导出”“引用”“文献管理软件导出”功能。
3. 将导出的题录、引用格式、RIS、BibTeX 或复制的摘要粘贴给 skill。
4. skill 再帮你整理成文献矩阵、研究脉络和综述段落。

## 4. 学校图书馆数据库怎么接入

学校图书馆数据库通常涉及机构权限，开源版只提供人工检索路径和导入入口。

你可以先通过学校图书馆进入数据库，下载或复制合法获得的题录、摘要、引用信息，再交给 skill 处理。skill 不会尝试绕过学校认证，也不会代替你下载受限全文。

## 5. 支持 Zotero、BibTeX、RIS 吗

支持。

推荐优先使用 Zotero、BibTeX 或 RIS，因为这些格式比手动复制更稳定，通常能保留作者、年份、题名、期刊、DOI、URL 等字段。

如果你使用 Zotero，可以先选中文献，导出为 BibTeX 或 RIS，再粘贴给 skill。具体示例见：

- [Zotero/BibTeX/RIS 导入示例](../examples/04-zotero-bibtex-ris-import.md)
- [复制即用提示词模板](./copy-paste-prompts.md)

## 6. 为什么要先生成文献矩阵

文献矩阵是为了减少假引用、空泛综述和逻辑混乱。

一篇文献综述至少要知道每篇文献的作者、年份、研究对象、研究方法、主要结论和局限。先做矩阵，再写综述，能让后面的研究脉络、研究空白和创新点更可靠。

## 7. 它会帮我分析“谁提出在先，谁参考了谁”吗

会，但会区分证据强弱。

如果文献元数据、参考文献列表或公开引用数据足够，skill 可以说明较明确的引用或承接关系。如果证据不足，它会根据发表时间、研究主题和方法相似性给出谨慎判断，并提醒你进一步核验。

推荐输出方式：

- 谁较早提出某个问题
- 谁在后续研究中扩展了研究对象
- 谁补充了方法或数据
- 哪些研究之间可能存在对比、承接或回应关系
- 哪些关系缺少直接引用证据，需要人工确认

## 8. 能帮我找创新点吗

可以。

skill 会基于文献矩阵和研究空白，给出 3-5 个可写方向，并说明：

- 这个方向为什么有空间
- 与这个创新点最接近的 1-3 篇论文
- 这些论文与创新点相近在哪里
- 用户可以从对象、场景、方法或数据上如何错开或推进
- 推荐使用的方法论
- 需要哪些数据或材料
- 适合本科、硕士还是期刊论文
- 可以放在论文哪个章节展开
- 风险和限制在哪里

它不会承诺某个创新点一定新、一定能发，也不会替代导师或作者的最终判断。

## 9. 怎么判断创新点会不会重复已有论文

v0.1.8 会为每个创新点标记最接近论文，并说明相近之处和可推进空间。

你需要重点核验：

- 最接近论文是否真实存在
- 相近之处是否来自题名、摘要、结论或用户提供的原文摘录
- 所谓创新点是否只是换了一个说法
- 推荐方法论是否能拿到数据或材料
- 如果相近论文来自公开来源候选文献，是否已经回到数据库或期刊官网核验

开源版只做文字化、表格化判断，不做自动相似度评分、创新点评分或可视化关系图。这些更适合 Pro 版。

## 10. 能创建研究领域知识地图吗

可以。

研究领域知识地图适合在你想快速理解一个方向时使用。它会输出：

- 1 个核心主张
- 3-5 个支持柱
- 2-3 个争议区
- 1-2 个前沿问题
- 初学者必读论文及推荐原因

如果文献来自公开来源候选结果，知识地图必须标注为“待核验版”。如果文献已经由用户导入并确认，才适合进一步转成正式综述和论文框架。

## 11. 输出文字会不会 AI 味很重

skill 默认会减少模板句和空泛表达，尽量写成更自然的中文学术表达。

它会避免过度使用这类句式：

- “具有重要意义”
- “随着时代的发展”
- “国内外学者进行了大量研究”
- “本文具有一定创新性”

但去 AI 化不等于改事实。作者、年份、期刊、结论和引用关系不能为了自然表达而被改写或虚构。

## 12. 本科、硕士、期刊论文模式有什么区别

三种模式的侧重点不同：

| 模式 | 重点 |
| --- | --- |
| 本科论文模式 | 结构清楚、概念稳妥、文献分类明确、创新点小而可完成 |
| 硕士论文模式 | 研究脉络、方法差异、研究空白、数据可行性和理论贡献 |
| 期刊论文模式 | 问题意识、文献争议、贡献表达、论证密度和期刊匹配 |

你可以在提示词中直接说：

```text
请按硕士论文模式输出。
```

## 13. 它能推荐期刊吗

可以给出期刊匹配思路和人工核验清单，但不会编造影响因子、录用率、版面费、审稿周期或收录信息。

推荐期刊时应核验：

- 期刊官网
- 最新征稿范围
- 最近发表文章主题
- 是否仍然正常出版
- 是否适合你的论文类型
- 是否需要版面费或其他费用
- 学校或单位是否认可该期刊

## 14. 会不会编造参考文献

不应该。

skill 的基本要求是：没有来源的文献不写入正式综述；字段不完整的文献进入“待核验文献”；无法确认的信息必须明确标注“需核验”。

如果用户要求直接生成正式参考文献，skill 应先检查来源字段，而不是凭空补齐作者、题名、期刊和年份。

## 15. 可以直接让它写完整论文吗

不建议。

更合理的用法是把论文写作拆成几个步骤：

1. 主题和关键词拆解
2. 文献检索和导入
3. 文献矩阵
4. 研究脉络
5. 研究空白
6. 创新点
7. 文献综述段落
8. 人工核验和修改

这样产出的内容更可靠，也更接近真实论文写作流程。

## 16. Pro 版会做什么

Pro 版适合放更重的数据处理和授权接口能力，例如：

- 授权万方接口接入
- 批量文献矩阵生成
- 文献关系图谱
- 引用链路增强分析
- 创新点评分和可行性排序
- 自动相似度评分、方法论适配评分和创新点可视化关系图
- 中文期刊推荐报告
- Word 文档导出
- 课题组或机构版模板

CNKI、维普、学校图书馆等数据库后续仍应以合规接入为前提：优先支持用户导出文件，只有在有官方授权接口时再做自动化接入。

## 17. 第一次使用应该怎么问

如果你已经有文献：

```text
使用 chinese-academic-paper-assistant。
我的论文主题是【主题】，论文类型是【本科论文/硕士论文/期刊论文】。
下面是我已经找到的文献资料，请先整理成文献矩阵，检查字段完整性，再分析研究脉络、研究空白和创新点。
不要编造文献，不要把待核验文献写进正式综述。

【粘贴文献资料】
```

如果你还没有文献：

```text
使用 chinese-academic-paper-assistant。
我的论文主题是【主题】，研究方向是【方向】，论文类型是【本科论文/硕士论文/期刊论文】。
我现在还没有整理好的文献。请先从公开来源查找候选文献，并生成 CNKI、万方、维普、学校图书馆和期刊官网的检索策略。
如果候选文献元数据足够，请生成待核验版文献矩阵和初步综述，但不要把它当成正式可引用综述。
```
