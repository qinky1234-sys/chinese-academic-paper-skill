# 新手使用文档

这份文档给第一次使用 `chinese-academic-paper-assistant` 的用户看。目标不是让 AI 直接替你写完整论文，而是先把真实文献整理清楚，再生成可核验的文献矩阵、综述段落、研究脉络和创新点。

## 适合谁

- 正在写本科毕业论文、课程论文、开题报告的学生
- 正在写硕士论文、博士论文、期刊论文的研究者
- 需要整理 CNKI、万方、维普、学校图书馆、Zotero 文献的人
- 需要把零散题录整理成文献矩阵、综述段落和研究空白的人

## 它能做什么

- 把用户导入的文献整理成文献矩阵
- 检查作者、题名、期刊、年份、摘要、关键词、DOI/链接等字段是否完整
- 分析文献共同点、研究脉络和可能的承接关系
- 总结当前研究空白，给出可写的创新点方向
- 生成更自然的中文文献综述段落
- 给出中文期刊推荐思路和人工核验提醒

## 它不能做什么

- 不编造参考文献
- 不绕过 CNKI、万方、维普、学校图书馆的登录、付费、验证码或权限限制
- 不承诺论文录用、查重结果或 AI 检测结果
- 不把字段不完整、来源不可追溯的文献直接写进正式综述

## 安装方式

把 skill 文件夹复制到你的 Codex skills 目录：

```powershell
mkdir $env:USERPROFILE\.codex\skills -Force
Copy-Item -Recurse .\chinese-academic-paper-assistant $env:USERPROFILE\.codex\skills\
```

如果你的 Codex 支持从 GitHub 安装 skill，可以使用这个路径：

```text
https://github.com/qinky1234-sys/chinese-academic-paper-skill/tree/main/chinese-academic-paper-assistant
```

## 第一次使用建议

不要一开始就说“帮我写论文”。推荐先准备 8-15 条文献题录，然后让 skill 先做文献矩阵。

每条文献最好包含：

- 作者
- 题名
- 期刊或来源名称
- 发表年份或日期
- 摘要
- 关键词
- DOI、URL 或数据库来源

如果你已经用 Zotero 管理文献，可以从 Zotero 导出 `BibTeX` 或 `RIS`。如果文献来自 CNKI、万方、维普或学校图书馆，优先使用数据库自己的“导出”“引用”“文献管理软件导出”功能。

## 推荐使用流程

1. 明确论文主题、研究方向和论文类型。
2. 导入你已有的文献题录、BibTeX、RIS、Zotero 导出内容或数据库导出内容。
3. 让 skill 先生成文献矩阵，不急着写正文。
4. 检查文献是否缺字段。
5. 如果没有文献或文献数量不够，再让 skill 从公开来源查找候选文献，并生成 CNKI、万方、维普和公开来源的补充检索策略。
6. 在文献矩阵基础上生成研究脉络、研究空白和创新点。
7. 最后再生成去 AI 化中文综述段落。

## 选择输出模式

v0.1.5 支持三种输出模式：

| 模式 | 适合用户 | 特点 |
| --- | --- | --- |
| 本科论文模式 | 本科毕业论文、课程论文 | 更重视概念清楚、分类清楚、创新点可完成 |
| 硕士论文模式 | 硕士论文、开题报告 | 更重视研究脉络、研究空白、方法和数据可行性 |
| 期刊论文模式 | 投稿论文、期刊论文初稿 | 更重视问题意识、论证密度、贡献表达和投稿匹配 |

可以这样说：

```text
请按本科论文模式输出。
请按硕士论文模式输出。
请按期刊论文模式输出。
```

## 最小可用提示词

```text
使用 chinese-academic-paper-assistant。
我的论文主题是【填写主题】，研究方向是【填写方向】。
论文类型是【本科论文/硕士论文/期刊论文】。
下面是我已经找到的文献资料，请先整理成文献矩阵，检查字段是否完整，再分析文献共同点、研究脉络、研究空白和可选创新点。
不要编造文献，不要把待核验文献写进正式综述。
```

## 常见问题

### 没有文献可以直接用吗

可以。skill 会先帮你生成检索关键词和数据库检索式，再根据公开来源或已配置接口补充候选文献。如果候选文献的作者、题名、来源、年份和链接等元数据足够，它可以先生成“待核验版”文献矩阵和初步综述。正式综述仍然建议建立在用户确认过的真实文献上。

### CNKI、万方、维普能自动接入吗

开源版不自动绕过这些网站的登录、付费、验证码或学校权限。推荐你先在对应网站合法检索和导出题录，再把导出内容交给 skill。

### 为什么要先做文献矩阵

文献矩阵可以减少假引用和空泛综述。只有先知道每篇文献的作者、年份、期刊、观点和局限，后面写出的综述才更接近真实论文写作流程。

### 输出的文字会不会 AI 味很重

skill 默认会减少模板句和空泛表达，但不会为了“像人写的”而改变事实、伪造引用或虚构个人经验。
