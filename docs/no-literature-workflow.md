# 没有文献时怎么用

这份文档给“现在手里还没有文献”的用户使用。它的目标是让你先得到一批可核验的候选文献、检索策略和初步综述框架，而不是让 AI 直接编出参考文献。

## 你会得到什么

当你没有文献时，`chinese-academic-paper-assistant` 可以先帮你产出：

- 关键词拆解
- CNKI、万方、维普、学校图书馆和期刊官网检索式
- 公开来源候选文献
- 待核验版首批文献主张表：作者+年份+一句话核心主张
- 待核验版共同假设分组、矛盾/张力标记和宏观背景图
- 待核验版文献矩阵
- 待核验版研究脉络
- 待核验版研究领域知识地图
- 待核验版研究空白
- 待核验版创新点建议
- 每个创新点对应的待核验最接近论文和推荐方法论
- 待核验版去 AI 化综述段落
- 待核验版完整版文字式文献综述，作为最后追加模块逐篇覆盖所有已进入候选表或候选矩阵的文献
- 中文期刊匹配思路和人工核验清单
- 下一步人工核验清单

注意：这些输出是为了帮你开始检索和判断方向。正式论文中的文献综述，仍然应该建立在你确认过的真实文献上。

## 第 1 步：准备最小信息

你至少需要提供：

| 信息 | 示例 |
| --- | --- |
| 论文主题 | 乡村振兴背景下基层数字治理研究 |
| 研究方向 | 公共管理、基层治理、数字政府 |
| 论文类型 | 本科论文、硕士论文、期刊论文 |
| 研究对象 | 县域、乡镇、社区、村庄、基层政府 |
| 你关心的问题 | 数字治理如何影响基层服务效率 |

如果你还没有明确题目，也可以提供一个大方向，让 skill 先帮你拆关键词。

## 第 2 步：复制这个提示词

```text
使用 chinese-academic-paper-assistant。

我的论文主题是【填写主题】。
研究方向是【填写方向】。
论文类型是【本科论文/硕士论文/期刊论文】。
研究对象是【填写对象，如大学生、企业、县域、社区、学校等】。
我现在还没有整理好的文献。

请先帮我完成从零查找文献的第一轮工作：
1. 拆解核心关键词、同义词、扩展词和排除词。
2. 生成 CNKI、万方、维普、学校图书馆和期刊官网可用的检索式。
3. 说明每组检索式适合查找哪类文献。
4. 从公开来源查找候选文献。
5. 如果候选文献元数据足够，请生成待核验版文献矩阵。
6. 基于候选文献，生成待核验版文献共同点、研究脉络、研究空白和创新点。
7. 为每个创新点标记 1-3 篇最接近论文、相近之处、可推进空间和推荐方法论。
8. 最后追加待核验版完整版文字式文献综述，不改变前面 1-7 步输出；所有已进入候选表或候选矩阵的文献都要逐篇对应写到，格式为“XXX学者在《xxxxx》一文中提出/指出/认为……”。

要求：
- 所有公开来源候选文献必须标注为“待核验”。
- 不要把待核验文献当作正式可引用文献。
- 不要绕过 CNKI、万方、维普、学校图书馆的登录、付费、验证码或权限限制。
- 不要编造作者、题名、期刊、年份、DOI 或研究结论。
```

## 第 3 步：先看关键词是否合理

第一轮输出里，优先检查关键词。

你需要看：

- 核心概念有没有偏题
- 研究对象有没有写清楚
- 有没有太宽泛的词
- 有没有遗漏政策词、方法词或场景词
- 有没有需要排除的无关方向

例如，“基层数字治理”可以拆成：

| 类型 | 示例 |
| --- | --- |
| 核心词 | 基层治理、数字治理、数字政府 |
| 对象词 | 乡镇政府、社区治理、村级治理、县域治理 |
| 方法词 | 案例研究、实证分析、问卷调查、政策文本分析 |
| 场景词 | 乡村振兴、公共服务、网格化治理、政务服务 |
| 排除词 | 企业数字化、平台经济、智慧城市硬件建设 |

如果关键词偏了，先让 skill 改关键词，不要急着生成综述。

## 第 4 步：用检索式去中文数据库核验

skill 会生成适合中文数据库的检索式。你需要手动去数据库里查：

- CNKI
- 万方
- 维普
- 学校图书馆数据库
- 相关期刊官网

推荐检索方式：

```text
主题 = "基层治理" AND "数字治理"
主题 = "乡村振兴" AND "数字治理" AND "基层"
篇名 = "基层数字治理" OR 关键词 = "基层数字治理"
主题 = "数字政府" AND "公共服务" AND "基层"
```

不同数据库语法可能不完全一样，复制后可以按页面提示调整。

## 第 5 步：导出或复制题录

查到相关文献后，优先导出这些字段：

| 字段 | 是否必要 |
| --- | --- |
| 作者 | 必要 |
| 题名 | 必要 |
| 期刊或来源 | 必要 |
| 发表年份或日期 | 必要 |
| 摘要 | 推荐 |
| 关键词 | 推荐 |
| DOI、URL 或数据库来源 | 推荐 |
| 基金项目、栏目、分类号 | 可选 |

如果数据库支持导出，优先选择：

- 引用
- 题录
- RIS
- BibTeX
- EndNote
- NoteExpress
- 文献管理软件导出

然后把导出的内容粘贴回 skill。

## 第 6 步：让 skill 生成正式矩阵

当你从中文数据库或期刊官网拿到题录后，用这个提示词：

```text
下面是我从 CNKI/万方/维普/学校图书馆/期刊官网导出的文献题录。
请先识别格式，再整理成正式文献矩阵。

要求：
1. 检查作者、题名、期刊/来源、年份、摘要、关键词、DOI/链接是否完整。
2. 将字段完整的文献标为“可进入正式综述”。
3. 将字段不完整或来源不清的文献标为“待核验文献”。
4. 不要补编缺失字段。
5. 不要把待核验文献写进正式综述。

【粘贴题录】
```

## 第 7 步：再生成综述和创新点

当正式矩阵里至少有 8-15 篇相对完整的文献后，再让 skill 写综述。

```text
请基于“可进入正式综述”的文献，生成一段中文文献综述。

要求：
1. 先概括这些文献的共同点。
2. 再说明研究脉络：谁较早提出，谁可能延伸或回应了前人的研究。
3. 没有引用证据时，不要直接说“谁引用了谁”。
4. 总结当前研究空白。
5. 给出 3-5 个适合我论文的创新点。
6. 每个创新点都要附 1-3 篇最接近论文、相近之处、可推进空间、推荐方法论、所需数据和核验状态。
7. 最后追加完整版文字式文献综述，所有已进入矩阵或候选表的文献都要逐篇对应写到。
8. 文字要自然，减少 AI 味，保留作者和年份引用标记。
```

## 输出结果怎么看

| 输出 | 怎么使用 |
| --- | --- |
| 检索关键词 | 用来继续查数据库 |
| 中文数据库检索式 | 复制到 CNKI、万方、维普或学校图书馆 |
| 公开来源候选文献 | 作为补充线索，不能直接当正式引用 |
| 待核验版矩阵 | 用来筛选方向，核验后再转正式矩阵 |
| 待核验版综述 | 只能作为草稿框架 |
| 待核验版完整版文字式文献综述 | 用来检查每篇候选文献如何进入综述，正式使用前必须逐篇核验 |
| 创新点建议 | 需要结合导师意见、数据条件和论文要求再筛选 |
| 最接近论文与方法论 | 用来判断创新点是否可能重复已有研究，以及下一步该用什么研究设计 |

## 常见误区

不要这样做：

```text
帮我找 20 篇参考文献并直接写文献综述。
```

原因是：如果不核验来源，容易出现假文献、错期刊、错年份或错结论。

更好的问法：

```text
请先根据我的主题查找公开来源候选文献，并生成中文数据库检索策略。
候选文献先标为待核验，不要写入正式综述。
等我核验并导入题录后，再生成正式文献矩阵和综述。
```
