# 示例 4：Zotero/BibTeX/RIS 导入教程

Zotero、BibTeX、RIS 是进入“一键生成文献综述”的推荐路径：导入 8 篇以上文献后，可直接生成完整综述和覆盖简表。

这个示例展示用户不知道如何把 Zotero 或数据库文献交给 skill 时，skill 应该怎样引导。

注意：下面的文献内容只用于演示格式，不代表真实参考文献。

## 示例输入

```text
我在 Zotero 里已经整理了一批关于“数字治理与基层公共服务”的文献。
我应该怎么导出给你？如果是 CNKI、万方、维普里的文献，也能这样导入吗？
```

## 示例输出

可以。你先把已有文献导出来，我会先处理你提供的资料；如果数量或字段不够，再补充检索候选文献。

### 一、Zotero 推荐导出方式

在 Zotero 里按这个路径操作：

1. 选中你要分析的文献。
2. 右键选择 `Export Items...`。
3. 优先选择 `BibTeX` 或 `RIS`。
4. 如果只是做文献矩阵和综述，通常不用勾选笔记和附件。
5. 导出后，把 `.bib` 或 `.ris` 文件内容发给我，或者直接上传文件。

选择建议：

- 如果文献主要来自 Zotero、Google Scholar、DOI 或英文数据库，优先用 `BibTeX`。
- 如果文献来自 CNKI、万方、维普、EndNote、NoteExpress 或学校图书馆，优先用 `RIS`。
- 如果数据库只支持复制题录，也可以直接粘贴，但请尽量包含作者、题名、期刊、年份、摘要、关键词和来源链接。

### 二、CNKI、万方、维普也可以接入

可以，但方式是“用户合法导出后导入”，不是自动绕过登录或付费页面。

推荐路径：

1. 在 CNKI、万方、维普或学校图书馆里检索并选中文献。
2. 使用网站自带的“导出”“引用”“文献管理软件导出”等功能。
3. 优先选择 `RIS`、`BibTeX`、`EndNote`、`NoteExpress`。
4. 尽量勾选摘要、关键词、DOI、来源链接。
5. 把导出的文本或文件交给我。

### 三、我收到后会做什么

我会先识别格式：

- `@article{...}` 这类内容按 BibTeX 解析。
- `TY  - JOUR`、`AU  -`、`TI  -`、`ER  -` 这类内容按 RIS 解析。
- 普通复制文本按题录字段整理。

然后我会检查每条文献是否具备这些字段：

| 字段 | 用途 |
| --- | --- |
| 作者 | 用于引用和研究脉络判断 |
| 题名 | 用于主题归类 |
| 期刊或来源 | 用于判断文献类型和可信度 |
| 发表年份或日期 | 用于判断谁提出在先 |
| 摘要或关键结论 | 用于综述和创新点分析 |
| DOI、URL 或数据库来源 | 用于追溯和人工核验 |

字段齐全的文献会进入正式文献矩阵。字段不全的文献会先放在“待核验文献”里，不会直接写进正式综述段落。

### 四、导入后的下一步输出

我会输出：

1. 已识别的导入格式
2. 成功解析的文献数量
3. 缺失字段清单
4. 可进入正式综述的文献
5. 需要补充或人工核验的文献
6. 文献矩阵
7. 文献共同点、研究脉络、研究空白和创新点

## 示例 BibTeX 输入片段

```bibtex
@article{zhang2024digital,
  author = {张三 and 李四},
  title = {数字治理背景下基层公共服务协同机制研究},
  journal = {公共管理研究},
  year = {2024},
  keywords = {数字治理; 基层治理; 公共服务},
  abstract = {文章讨论数字治理工具在基层公共服务协同中的作用机制。},
  url = {https://example.com/article}
}
```

## 示例 RIS 输入片段

```ris
AU  - 赵六

```
