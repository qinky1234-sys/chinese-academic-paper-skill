# 推广执行清单

这份清单用于把 `chinese-academic-paper-skill` 推广到中文 GitHub 相关社区，同时避免被看成垃圾广告。

## 已准备素材

- GitHub 仓库首页：README 已改成产品首页。
- 一键综述示例：`examples/10-one-click-literature-review.md`
- 示例案例：`examples/05-full-chinese-demo-case.md`
- 三种输出模式：`examples/06-output-modes-demo.md`
- 新手文档：`docs/getting-started.md`
- 复制即用提示词：`docs/copy-paste-prompts.md`
- 中文社区推广帖：`promotion/github-chinese-community-post.md`

## 推荐发布位置

1. GitHub 中文论坛的“开源分享”分类
   - 适合发完整介绍帖。
   - 标题直接说明项目用途，不要写“自动写论文”。

2. GitHub 中文社区类项目展示网站
   - 适合提交项目链接，等待编辑或系统收录。
   - 重点突出“一键生成完整文献综述”“中文学术写作”“不编造引用”。

3. GitHub Topics
   - 通过 topics 提高搜索发现概率。
   - 推荐 topics：`academic-writing`, `literature-review`, `chinese-academic-writing`, `cnki`, `wanfang`, `zotero`, `codex-skill`, `research-assistant`。

4. 自己仓库的 Issues
   - 创建一个中文反馈征集 Issue。
   - 用来接收用户案例、bug、数据库导入问题和功能建议。

## 发布时避免的说法

- 不要说“自动写论文神器”。
- 不要说“保证通过查重/AI 检测”。
- 不要说“自动下载 CNKI/万方/维普全文”。
- 不要暗示可以绕过数据库付费、登录或学校权限。

## 推荐卖点

- 先整理真实文献，再写综述。
- 上传 8 篇以上文献，一键生成完整文献综述和文献覆盖简表。
- 用户没有文献时，也可以先查找公开候选文献并生成待核验版综述。
- 支持 Zotero、BibTeX、RIS、CNKI、万方、维普题录导入。
- 支持本科、硕士、期刊论文三种输出模式。
- 明确标注文献核验边界，减少假引用风险。

## 后续跟踪指标

- GitHub Stars
- Issues 数量
- 用户是否能独立完成安装
- 用户是否能理解“待核验候选文献”
- 哪种模式使用最多：本科、硕士、期刊
- 用户是否需要 Word 导出、批量分析或万方授权接入
